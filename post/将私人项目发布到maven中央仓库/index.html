将个人项目发布到maven仓库，使得个人项目可以通过maven引入

环境

操作系统：Mac OS

maven版本：3.6

项目地址：GitHub

第一步：maven项目发布申请

1.到官网创建一个账号

地址：https://issues.sonatype.org/secure/Dashboard.jspa

2.创建一个issue

如上为必填字段，还有一个描述就写你项目功能吧，填完后点击 创建 。

如上是创建完成后的issue。

安装gpg

gpg是用来生成密钥并将公钥上传到密钥服务器，我是Mac系统，安装的是gpg，Windows下使用gpg4win来进行配置。下载地址 https://www.gpg4win.org/get-gpg4win.html

# mac安装gpg
brew install gpg

# 查看是否安装成功
gpg --version

# 生成密钥,生成时要填入用户名，邮箱，密码，密码在打包发布时要用到
gpg --gen-key

# 查看公钥
gpg --list-keys

# 将生成的公钥发布到服务器
gpg --keyserver hkp://pool.sks-keyservers.net --send-keys 你的公钥

# 查看公钥是否发布成功
gpg --keyserver hkp://pool.sks-keyservers.net --recv-keys 你的公钥

打包发布

现在，一切准备工作都做好了，终于可以打包发布到maven仓库，奥利给！

打包命令：

1

mvn clean deploy -P release verify '-Dgpg.passphrase=gpg密钥密码'

进入maven网站发布项目

进入该网站，用户名密码同样是最开始时注册的：https://oss.sonatype.org/#stagingRepositories

如上所示先点击close按钮，然后可以到下面的active面板看进度，得一会儿，close完成后才能点release按钮，完成后算是发布成功了，但是要在maven中央仓库能搜到还得几个小时，不过你可以自己直接引入试试，看看发布的包对不对

最后

此时包算是发布完成了，我们要到我们创建的issue上加个评论说一下，工作人员好把我们把这个issue关掉。

遇到问题

我们做一件事，不都是一帆风顺，总会遇到一些预料之外的事情，以下就是我遇到的坑，如果你也遇到了，希望能帮你避过坑，一帆风顺。

运行打包发布命令时，遇到这个问题：gpg: signing failed: Inappropriate ioctl for device

从错误上看，很明显，gpg加密解密有问题，经过一番百度,说是gpg无法弹出窗口，我先是在我的 .bash_profile(mac系统配置文件)文件增加了 如下配置

1

export GPG_TTY=$(tty)

然而并没有啥用

然后我在 .gnupg 文件下增加了如下两个配置文件和内容

gpg-agent.conf

1

allow-loopback-pinentry

gpg.conf

1
2

use-agent
pinentry-mode loopback

问题解决，打包成功。
